# Backend Local Docs

Use this folder for backend-specific notes that should stay close to implementation.

## Current Files

- `local-development.md` for local runtime, database, Docker, and test details
- `frontend-integration-guide.md` for endpoint, entity, and frontend data-flow guidance

## Guidance

- Keep shared product or contract decisions in `docs/shared/`.
- Keep backend-only runtime and implementation notes here.
- When backend architecture changes, update both this folder and the shared docs if the change affects other workspaces.
